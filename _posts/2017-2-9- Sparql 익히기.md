# JUMPING RIGHT IN : Some Data and Some Queries

###SPARQL

**Sparql**은 *SPARQL Protocol and RDF QUERY LANGUAGE* 재귀적 약어이다. HTML, XML, CSS와 마친가지로 그 표준제약은 World Wide Web Consortium에서 담당하고 있다.
"RQL" 부분의 이름에서 알 수 있듯, SPARQL은 RDF(Resource Data Frame)라는 데이터 모델을  쿼리(Query)하는데 쓰이는 언어이다.

#Sparql로 쿼리(Qeury)할 데이터

SPARQL 언어의 탐색 대상이 되는 RDF는 그 자체로서 데이터 형식(format)이 되진 않는다. RDF 자체는 데이터 모델을 일컫는다. RDF 모델에서는 "Triples"이라고 일컬어 지는 세 개의 Statements들로 이루어져 있다. Triple의 세 부분은 Subject, Predicate, Object인데, 각 부분의 이름대로의 기능을 수행한다. 즉, Subject는 리소스의  Identifier, Predicate는 Property Name, Object는 Property Value에 해당된다.

```
## filename: ex002.ttl

@prefix ab: <http://learningsparql.com/ns/addressbook#> .

ab:richard ab:homeTel "(229) 276-5135" .
ab:richard ab:email   "richard49@hotmail.com" .

ab:cindy ab:homeTel "(245) 646-5488" .
ab:cindy ab:email   "cindym@gmail.com" .

ab:craig ab:homeTel "(194) 966-1505" .
ab:craig ab:email   "craigellis@yahoo.com" .
ab:craig ab:email   "c.ellis@usairwaysgroup.com" .
```
위 파일은 Turtle RDF format의 예시이다. 이 예시는 주소록 데이터를 담고 있는데, "Richard의 homeTel 번호는 "(229) 276-5135" 이다"라는 문장을

>ab:richard ab:homeTel "(229) 276-5135" .

라는 것으로 나타내고 있다. 또한


>ab:craig ab:email   "craigellis@yahoo.com" .

>ab:craig ab:email   "c.ellis@usairwaysgroup.com" .

에서 볼 수 있듯, RDF에서는 하나의 Resource에 대해 똑같은 Property에 여러개의 Value가 할당되는 것은 문제 없다.

보통 문장에서 '.(period)'으로 문장을 끝내 듯, Turtle에서도 문장의 끝은 '.'로 표현한다.  '.'이전의 공백(스페이스바)는 상관이 없지만 가독성을 위해 추가하는 경우도 있다. 코드의 주석처리는 '#'를 사용해서 표시한다.

> @prefix ab: <http://learningsparql.com/ns/addressbook#> .

이제 위 구문에 주목하자. 이 구문은 prefix "ab"가  *<http://learningsparql.com/ns/addressbook#>* 이라는 URI를 나타낸다고 말하고 있다. RDF에서는 Subject와 Predicate는 필히 특정한 Namespace로 지정되어야 한다. 데이터의 모호성(ambiguity)을 방지하기 위함인데, 그 Namespace는 URI로 나타내어진다. "@Prefix"는 일일이 URI을 적어야 하는 수고를 덜어주는 역할을 한다.

 URI는 Uniform Resource Identifier의 약자로, 사용자가 무엇을 찾도록 도와주는 Locator 역할을 한다(WebPage나 Identifier의 역할과 같다). 예를 들어,  <http://learningsparql.com/ns/addressbook#>는 Unique한 URI이다. 이때, URI는 통상의 URL처럼 생겼지만, URI가 지정하는 곳에는 WebPage가 실제로 있을 수도 있고, 없을 수도 있다. URI의 주(primary) 역할은 '무언가(SomeThing)'를 지칭할 고유의 이름을 제공하기 위함이지,  사용자의 웹브라우저가 가야 할 웹 페이지를 사용자에게 알려주는 역할이 주된 것은 아니다.

# 데이터 쿼리(Query)하기
SPARQL로 하는 질의(쿼리)는 주로 다음과 같이 말한다
:	**"난 '이러한' 조건을 만족하는 데이터로 부터 정보의 부분들을 불러오고 싶어!"**
SPAQRL에선 이러한 조건들을 'Triple들의 패턴'으로 나타낼 수 있다. 그 형식은  RDF의 Triple들과 유사하지만, SPARQL의 Triple에선 '변수(Variable)'이 필요하다.

다음은 SPARQL로 작성된 질의문의 예시이다.

```
# filename: ex003.rq
# 참고로, SPARQL의 확장자는 .rq를 쓰도록 권장되고 있다.
PREFIX ab: <http://learningsparql.com/ns/addressbook#>

SELECT ?craigEmail
WHERE
{ ab:craig ab:email ?craigEmail . }
```
다음은 SPARQL의 질의 내에서 보이는 Triple Pattern이다
>ab:craig ab:email ?craigEmail .

RDF의 Triple과 마찬가지로, *S,P,O*의 구조를 가지고 있다. 위의 SPARQL의 Triple의 경우 *ab:craig*, *ab: email*가 각각 S,P에 해당되고, 변수의 역할을 하는 *?craigEmail*이 O에 해당된다. 또한 RDF와 마찬가지로 '.(period)'로 끝나는 Triple 문장을 갖는다. 질의할 Triple의 집합 중, 마지막 Triple엔 '.'을 쓰지 않아도 된다. 즉 Triple이 하나 뿐이 있을 땐 그 Triple 끝에 '.'표시를 하지 않아도 되는데, 가독성을 고려해 웬만하면 하도록 하는 것이 좋다. 또한 PREFIX, SELECT, WHERE와 같은 구절은 모두 대문자로 쓰여져 있지만, 사실 SPARQL은 대/소문자 구분을 하지 않는다.

위의 Triple Pattern이 질의 하고 있는 것을 알아보자. 이 질의문은 *ab:craig* 라는 리소스와 중 *ab:email*의 속성(Property)를 갖는 모든 값, 즉 데이터들을 불러오라는 뜻이다. 일반 언어로 표현하자면, "크레이그씨의 이메일 주소를 알려주세요!"라고 말하는 것과 마찬가지 이다.

그렇다면 이제, 내 컴퓨터에 저장된 .ttl 따위의 RDF 데이터를 직접 .rq 확장자의 SPARQL 언어를 사용해 질의한 결과를 표시해야한다. 이를 위해선 SPARQL 엔진( 혹은 SPARQL 프로세서)이 필요하다. *Learning With Sparql"에서 쓰는 Sparql 엔진은 자바 기반의 ARQ이다. ARQ은 Apache Jena FrameWork의 일부인데, 설치법과 사용법은  다음과 같다.

1. 이 [링크](https://jena.apache.org/download) 에서 "apache-jean-*.zip"파일을 다운 받으면 된다.
2. 다운받은 .zip파일의 압축을 풀어 임의의 디렉토리에 저장해 놓는다. 필자의 경우 /Users 디렉토리에 저장해 놓았다.
3. Linux/Mac에선 Terminal  상에서 다음과 같은 명령어를 쳐야 한다.
>export JENAROOT= *the directory you downloaded Jena to*
>export PATH=$PATH:$JENAROOT/bin
4. 그러고 나서 다음과 같은 커맨드를 쳐보자

```
filename: ex004.txt
arq --data ex002.ttl --query ex003.rq
```
위의 커맨드를 입력할 경우 질의 결과가 나타날 것이다.
